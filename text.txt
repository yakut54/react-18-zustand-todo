«17 — синхронный мир, ReactDOM.render; 18 — Concurrent Mode/Rendering, createRoot, новые хуки и автоматический батчинг». useTransition и useDeferredValue

create — фабричная функция Zustand которая:                                                                                                                        
  1. Принимает инициализатор (или функциональную композицию middleware с инициализатором внутри)                                                                                    
  2. Создаёт хранилище вне React-дерева                                                                                                                              
  3. Возвращает хук, который через useSyncExternalStore подписывает компонент на это хранилище
                                                                                                                                                                     
  useSyncExternalStore — встроенный React хук для подписки на внешние хранилища. Появился в React 18. Гарантирует консистентность стейта при concurrent rendering —
  именно поэтому Zustand безопасен с React 18+.

  Каждый middleware получает set от предыдущего, оборачивает его своей логикой и передаёт дальше. create в конце получает готовую цепочку.




  Оптимистичный апдейт — это когда UI обновляется сразу, 
  не дожидаясь ответа сервера. Мы предполагаем что запрос успешно выполнится. 
  Если сервер вернул ошибку — откатываем состояние назад. Улучшает perceived
  performance — приложение кажется быстрее.     


   Automatic Batching — React 18 объединяет несколько setState в один ререндер везде: в таймерах, промисах, нативных обработчиках. В React 17 это работало
  только в JSX-обработчиках (onClick, onChange). Результат — меньше лишних ререндеров без каких-либо изменений в коде.

  ● ▎ Reconciliation — это когда React сравнивает старый и новый Virtual DOM и обновляет в реальном DOM только то что изменилось. key нужен чтобы React понимал какой элемент списка есть какой —        
  ▎ особенно при удалении, добавлении и сортировке. index как key ломает это — если удалить первый элемент, все индексы сдвинутся и React подумает что изменились все элементы.


    ▎ Virtual DOM — это JS объект, легковесная копия реального DOM. При изменении стейта React строит новый VDOM, сравнивает его со старым (reconciliation), находит разницу и применяет только нужные
  ▎ изменения в реальный DOM. Это быстрее чем обновлять реальный DOM напрямую, потому что операции с JS объектом дешевле чем операции с DOM.


    ▎ useEffect запускается после того как браузер отрисовал изменения. useLayoutEffect — до отрисовки, синхронно блокирует браузер. Нужен когда надо измерить или изменить DOM до того как пользователь
  ▎ его увидит — иначе будет визуальный "прыжок". В 95% случаев нужен useEffect.

   ▎ useMemo мемоизирует результат вычисления — используй когда вычисление дорогое или нужна стабильная ссылка на объект/массив. useCallback мемоизирует функцию — используй когда передаёшь её в
  ▎ дочерний компонент обёрнутый в React.memo или в зависимости хуков. Без реальной проблемы с производительностью — не используй.
   Оба нужны чтобы не создавать новую ссылку при каждом рендере.


  ▎ React.memo пропускает ререндер если пропсы не изменились (сравнение через ===). Не поможет если передаёшь объекты или функции без мемоизации — каждый рендер создаёт новую ссылку и React.memo 
  ▎ думает что пропсы изменились. Решение — useCallback для функций, useMemo для объектов.

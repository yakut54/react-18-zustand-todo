create — фабричная функция Zustand которая:                                                                                                                        
  1. Принимает инициализатор (или функциональную композицию middleware с инициализатором внутри)                                                                                    
  2. Создаёт хранилище вне React-дерева                                                                                                                              
  3. Возвращает хук, который через useSyncExternalStore подписывает компонент на это хранилище
                                                                                                                                                                     
  useSyncExternalStore — встроенный React хук для подписки на внешние хранилища. Появился в React 18. Гарантирует консистентность стейта при concurrent rendering —
  именно поэтому Zustand безопасен с React 18+.

create внутри делает три вещи:                                                                                                                                     
                                                                                                                                                                   
  1. Вызывает твой инициализатор — (set, get) => ({...}) — и получает начальный стейт.                                                                               
   
  2. Создаёт хранилище вне React — объект с getState, setState, subscribe.                                                                                           
                                                            
  3. Возвращает хук useFilterStore — внутри вызывает useSyncExternalStore из React, который подписывает компонент на хранилище и триггерит ререндер при изменениях.

  это паттерн decorator — каждый слой добавляет поведение не меняя предыдущий.


  Функциональная композиция — f(g(h(x))):

  devtools( persist( (set) => ({...}) ) )

  Читается изнутри наружу:
  1. (set) => ({...}) — твой стор
  2. persist(...) — оборачивает set, добавляет localStorage
  3. devtools(...) — оборачивает снова, добавляет DevTools

  Каждый middleware получает set от предыдущего, оборачивает его своей логикой и передаёт дальше. create в конце получает готовую цепочку.

«Автоматический батчинг в React 18 — это механизм, при котором React сам группирует несколько setState (в таймере, fetch, Promise и т.д.) в один ререндер, чтобы не дергать DOM лишний раз.»

«Чем React 18 принципиально отличается от 17?»
— отвечаешь:
«17 — синхронный мир, ReactDOM.render; 18 — Concurrent Mode/Rendering, createRoot, новые хуки и автоматический батчинг». useTransition и useDeferredValue

Лучше возвращать новый объект с явными полями, чем использовать as User.
as User не валидирует данные, а только отключает часть проверки типов.
Если структура data.user может измениться или прийти лишнее/неполное поле, явное маппирование надёжнее и понятнее.
